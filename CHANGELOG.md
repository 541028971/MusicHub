# MusicHub 更新日志

## 版本 v0.1.0
**日期：** 2026年3月8日

### 🎨 视觉与 UI 增强
- **霓虹 Logo 修复**：恢复了原始的 "MusicHub" Logo，采用纯白核心和多层绿色霓虹光晕。添加了“呼吸”脉冲动画，让界面更生动。
- **推荐菜单图标**：在“精选”导航栏添加了时尚的 'Sparkles' 图标。
    - **颜色**：自定义金色 (#FFD700)
    - **尺寸**：放大至 18x18 以更加突出。
    - **完美对齐**：重构导航链接，使用 **Flexbox (`align-items: center`)**，确保图标和文字在视觉上完美居中。
- **玻璃拟态组件**：优化了侧边栏按钮和播放器元素，增加了毛玻璃效果和更精细的阴影深度。
- **主题一致性**：将主要强调色锁定为指定的 "MusicHub Green" (#00C78A)。

### 🎵 播放器性能优化
- **交互式进度条**：
    - **即时拖动**：实现了零延迟播放跳转。在手动拖动期间动态禁用 CSS 过渡，以获得极佳的跟随感。
    - **流媒体逻辑**：实现了一个自定义的 Django 媒体服务器，支持 **HTTP Range 请求** (Partial Content)，支持音频分段加载。解决了点击进度条会导致播放重置为 0 的问题。
- **悬浮时间显示**：
    - 引入了随进度条滑块移动的实时时间显示。
    - **边缘保护**：实现了边缘限制逻辑，确保时间显示在进度条两端不会超出屏幕边界。
- **体验细节**：
    - 拖动进度条时禁用全局文本选择，防止意外高亮文本。
    - 拖动时光标锁定为“抓取”状态。

### 🛠️ 后端与基础设施
- **媒体路由**：重构了 URL 模式，通过 `serve_media` 专门处理大音频文件的缓冲。
- **路径安全**：增强了媒体资源的路径规范化和访问安全。
- **代码库清理**：移除了一系列临时调试脚本，保持生产环境整洁。

## 版本 v0.2.0
**日期：** 2026年3月9日

### ❤️ 全站红心“点亮”系统与乐库增强
- **全域同步交互**：实现了核心红心收藏功能的完整逻辑。在首页、乐库、搜索结果、歌单详情或播放器中点击红心，全站所有位置的同一歌曲状态将瞬间同步。
- **乐库与搜索系统升级**：
    - **新列添加**：为 Library 和 Search 的歌曲列表新增了专用的 "Like" 列，对齐了 6 列网格布局，确保一致性。
    - **后端深度集成**：进入页面时后端会自动查询并匹配当前用户的收藏记录，实现加载即点亮。
- **视觉反馈优化**：
    - **普通/悬停**：未点亮时仅为轮廓线，悬停时轮廓变红并伴随缩放动画。
    - **点亮状态**：成功点亮后显示实心红心，并伴随全站动态通知提醒。

### 📂 全新歌单管理系统
- **添加到歌单弹窗**：在播放器右侧及歌曲卡片菜单中新增了“添加到歌单”入口。
- **交互设计**：实现了一个居中的玻璃拟态弹出窗口，支持查看“我喜欢的音乐”及用户创建的所有歌单。
- **后端接口**：新增了 JSON API 用于异步获取歌单列表和管理歌曲添加逻辑。
- **动态更新**：歌单列表按 ID 排序，支持实时刷新侧边栏计数。

### 🎨 导航与 UI 精简
- **导航标签架构优化**：
    - **极简主义**：移除了冗余的 "VIP" 标签，暂时隐藏了未开放的 "Top Charts" 和 "Artists"。
    - **视觉聚焦**：非活动状态的导航标签（如 Featured, Playlist Square 等）透明度调至 0.45，通过明暗对比突出当前页面。
- **组件细节打磨**：
    - **艺术家页面**：缩小了头像尺寸，并为背景增加了高不透明度的“毛玻璃”（Frosted Glass）纹理，提升了高级感。
    - **对比度校准**：提升了播放列表中专辑名称和序列号的亮度，使其与歌曲名称保持视觉平衡。

### 🛠️ 性能与代码修复
- **CSS 冲突消除**：彻底清理了 `discovery.css` 中与全局红心样式冲突的旧代码，解决了悬停 Bug。
- **脚本健壮性**：重写了全局同步选择器和侧边栏折叠逻辑，使用了更具兼容性的变量声明，解决了 Linter 警告。

## 版本 v0.3.0
**日期：** 2026年3月10日

### 🎨 UI 细节打磨与视觉统合
- **操作图标全站统合**：
    - 将“乐库 (Library)”、“播放列表详情”及“右键菜单”中的操作按钮（下载、添加、评论、删除）全部统一为 **Feather 线条美型样式**。
    - 统一了图标笔触粗细 (`stroke-width: 2`) 与尺寸，视觉感受更加轻盈、一致。
    - **特别还原**：根据用户反馈，首页卡片上的下载按钮保留了原有的实心视觉风格。
- **高级按钮交互效果**：
    - **发光系统**：为不同功能按钮设计了定制化的悬停发光色（下载-翠绿、添加-天蓝、删除-艳红、评论-纯白）。
    - **极简胶囊**：彻底去除了歌曲行操作按钮的背景“底座”，实现完全透明的玻璃态感，仅在悬停行时呈现微弱磨砂感。
    - **悬停反馈**：去除了歌单列表中会导致边缘裁剪的缩放效果，改为强化背景和边框的亮度反馈。
- **布局微调**：
    - **Play Now 图标**：将菜单中的“立即播放”图标放大至 18px，提升点击精度。
    - **菜单精简**：从歌曲右键菜单中移除了低频使用的“分享”选项，让界面更聚焦。

### 🎵 播放与下载体验增强
- **下载逻辑重构**：
    - 废弃了跳转中间页的旧模式，实现了**强制直接下载**逻辑。
    - 利用 `Blob` 技术确保点击后直接唤起浏览器的下载程序，保持页面不跳转、不闪动。
- **极简 SPA 导航优化**：
    - 修复了单页应用（SPA）模式下的切页 Bug。
    - **状态恢复**：解决了返回首页或进入乐库时，界面元素意外变淡或由于透明度未重置导致的显示异常。
    - **进度条跳转优化 (Seeking Fix)**：
    - **逻辑重构**：将真实的音频跳转指令从“鼠标按下”移至“鼠标抬起”，彻底解决了连点产生的“二重跳转”导致的破音和卡顿。
    - **视觉先行**：按下鼠标时 UI 立即跟随，但音频保持平滑播放，直到确定位置后再切过去，提升了操控的精度。
    - **性能调优**：优化了后端 `django` 媒体服务器的读取策略，平衡了分块大小与加载速度，确保大文件跳转秒开。

### 🛠️ 代码审计与环境清理
- **备份清理**：全量删除了项目开发过程中产生的 `.v6-backup`, `.v2-backup` 以及 `.pre-heart-fix` 等冗余备份文件，保持目录清爽。
- **JS 健壮性修复**：
    - 修复了 `loadAndPlay` 函数在部分页面参数传递不全导致的播放失败或卡死问题。
    - 解决了选择器由于大小写不匹配导致的 Hover 样式失效 Bug。
    - 移除了 `library.html` 中多处冗余的内联样式，交由全局 CSS 统一管理。
    - **工具保留**：保留了数据初始化和测试脚本，方便后续开发维护。

### 🛠️ 自动化媒体管理补丁
- **文件名标准化 (Auto-Renaming)**：
    - 在 `models.py` 中引入了自动命名管家，确保存储在服务器上的所有歌曲和封面都以 `{ID}_歌名` 格式命名，即使是首次上传也能完美获取 ID 编号。
    - **冗余清理逻辑**：实现了物理文件的全生命周期管理。当用户在后台修改、替换封面或删除歌曲时，服务器硬盘会自动同步清理无用的旧文件，防止存储空间被无效的“电子垃圾”堆满。

## 版本 v1.0.0 (Official Release - Project Aurora)
**日期：** 2026年3月16日

这是 MusicHub 的里程碑版本，标志着从基础原型到全功能**沉浸式音乐交互平台**的进化。我们将整个项目命名为 **"Project Aurora"**。

### 🚀 沉浸式 SPA 2.0 交互引擎
- **全站无缝导航**：重构了 `navigatePage` 核心，实现了 100% 无刷新的 SPA 体验。支持地址栏同步、动态组件挂载及脚本的重新初始化。
- **智能历史管理 (Popstate)**：完美适配浏览器后退/前进按钮，能够智能处理“收起弹出层”与“页面回退”的优先级逻辑。
- **极简搜索系统**：
    - **动态清除 (X)**：新增搜索框实时清除按钮，伴随交互式红/白色彩切换效果。
    *   **视觉占位符**：搜索框聚焦时占位符透明度平滑降低，减少视觉干扰。
    - **高亮搜索词**：搜索结果中，匹配的关键词会呈现 **Vibrant Cyan-Blue** 发光高亮效果。
- **全局 Header Back**：为所有深层页面提供了自定义的后退按钮。

### 🎵 深度互动的播放内核 (Project Vinyl)
- **模拟黑胶播放器 (PDV)**：
    - **物理仿真系统**：在播放详情页引入了转动的黑胶唱片与**动态唱针臂**，唱针会随播放状态自动落针/抬起。
    - **磨砂玻璃控制台**：播放器采用侧边三段式（控制区、唱片区、列表/词库区）布局，呈现极致的层次感。
- **智能滚动式歌词**：
    - **毫秒级同步**：实现了基于 `[mm:ss.xx]` 标准的歌词对齐，当前行会变绿并伴随 1.15 倍缩放与外发光效果。
    - **点击即跳转 (Line Seeking)**：直接点击歌词行即可控制进度条跳转至对应片段。
    - **动态模糊遮罩**：歌词容器上下边缘采用 `linear-gradient` 物理遮罩，实现优雅的淡入淡出。

### 🎨 极致的 UI/UX 美学规范
- **全站 Aurora 滚动条**：废弃了系统原生滚动条，设计了**霓虹绿 (Aurora Green)** 悬浮感滚动条，支持 Hover 亮度增强和自动隐藏。
- **歌曲标题跑马灯 (Marquee)**：针对卡片、列表中的超长歌名，鼠标悬停时会自动左右平滑滚动，确保信息完整性且不破坏布局。
- **排行榜勋章系统**：为 "Top Views" 榜单前三名设计了金、银、铜牌视觉效果，并支持 1-9 名的高级排版。
- **智能侧边栏 (Dynamic Sidebar)**：
    - **状态固化**：侧边栏支持“我创建的”与“我收藏的”歌单分组折叠，并记录折叠状态。
    - **实时计数**：歌单后的数字会随收藏操作即时跳动。

### 💬 社区互动与社交生态 2.0
- **多层级评论架构**：
    - **引用回复**：支持针对特定评论的二级嵌套回复，并在回复框中显示引用原句的引用块。
    - **热门排序**：评论区自动将高赞评论（Hot Comments）置顶。
- **交互式确认系统 (HubDialog)**：
    - 使用自定义的 **HubToast**（极简通知）和 **HubConfirm**（玻璃态确认框）替代了浏览器原生 Alert。
- **收藏同步网络**：全站“红心”已连接至实时同步网络，在任意位置点击，关联歌曲的状态将在 50ms 内瞬间对齐。

### 👤 个人成长与资产管理
- **全能设置中心**：支持修改个人基本信息（生日、地区、签名），以及头像的上传更新。
- **自研裁剪引擎 (Hub Cropper)**：在上传头像和歌单封面时，会自动唤起内置的**圆形/方形裁剪引擎**，确保在 UI 中展示完美。
- **物理资产清理**：引入了 Django 信号监听，当后台删除歌曲或替换图片时，服务器硬盘会自动销毁旧文件，真正做到“零垃圾”堆积。

### 🛡️ 开发者效能与同步标准
- **数据库隔离标准**：彻底清理了 Git 对 `db.sqlite3` 的跟踪，转而使用轻量级的 `music_data.json` 导出同步。
- **后台管理系统增强**：
    - **Dashboard 2.0**：集成 Chart.js，为管理员提供可视化用户增长和内容分布图表。
    - **快速返回机制**：点击管理后台侧边栏底部的 `Back` 按钮可一键返回前台。
